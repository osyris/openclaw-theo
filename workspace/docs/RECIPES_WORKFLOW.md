# RECIPES_WORKFLOW.md

Правила для каталога рецептов, чтобы новые рецепты добавлялись быстро, одинаково и без риска что-то сломать.

---

## 1. Главный принцип

Для рецептов мы больше **не изобретаем велосипед каждый раз**.

Новый рецепт = это **не правка HTML-страницы**, а **добавление одной новой записи в `recipes.json`**.

Страница `pages/recipes/index.html` должна оставаться в роли витрины:
- показывает рецепты
- фильтрует
- ищет
- открывает карточку

А сами рецепты живут отдельно в данных.

---

## 2. Где что лежит

```text
/data/workspace/pages/recipes/
  index.html         ← сама страница каталога
  recipes.json       ← все рецепты
  img/               ← картинки рецептов
```

### Правило
- `index.html` — трогать только если меняется логика или внешний вид всей страницы
- `recipes.json` — трогать, когда добавляем/редактируем рецепты
- `img/` — хранит изображения рецептов

---

## 3. Как правильно добавлять новый рецепт

### Правильный путь
1. Подготовить текст рецепта
2. Подготовить картинку
3. Добавить **один объект** в `recipes.json`
4. Проверить, что картинка открывается
5. Проверить, что рецепт находится через поиск и фильтры

### Неправильный путь
- не вписывать новый рецепт прямо в HTML
- не делать отдельную страницу под каждый рецепт
- не хранить текст рецепта в заметках “где-то потом вставлю”
- не менять структуру рецепта от случая к случаю

---

## 4. Стандарт одного рецепта

Каждый рецепт должен иметь одинаковую структуру.

### Обязательные поля
```json
{
  "title": "Текст на русском",
  "titleEn": "English title",
  "category": "dinner",
  "time": "20 мин",
  "servings": 2,
  "difficulty": "easy",
  "description": "Короткое понятное описание",
  "image": "img/example.jpg",
  "ingredients": [
    "ингредиент 1",
    "ингредиент 2"
  ],
  "steps": [
    "шаг 1",
    "шаг 2"
  ],
  "favorite": false
}
```

### Необязательные поля
Можно добавлять, если уместно:
```json
{
  "calories": "200 ккал",
  "protein": "18 г белка"
}
```

---

## 5. Правила по полям

### `title`
- основное название на русском
- коротко и понятно
- без лишних эмодзи

### `titleEn`
- аккуратное английское название
- нужно для красоты и единообразия карточек

### `category`
Используем только существующие категории страницы:
- `breakfast`
- `lunch`
- `dinner`
- `salad`
- `soup`
- `snack`
- `dessert`
- `drink`

Если хочется новую категорию — сначала отдельно решаем, добавляем ли её в интерфейс.

### `time`
Пишем единообразно:
- `15 мин`
- `40 мин`
- `1 ч`
- `30 мин + 4ч пропитка`

### `servings`
- только число
- без слова "порции"

### `difficulty`
Только из списка:
- `easy`
- `medium`
- `hard`

### `description`
- 1 короткое описание
- без слишком длинного текста
- задача описания: быстро понять, что это и почему рецепт хороший

### `image`
- путь только относительный
- всегда в формате: `img/filename.jpg`

### `ingredients`
- каждый ингредиент отдельной строкой
- не объединять всё в один абзац
- допустимы служебные разделители вроде `— Соус: —`, если это реально улучшает читаемость

### `steps`
- каждый шаг отдельной строкой массива
- коротко, ясно, по делу
- не превращать один шаг в полстраницы

### `favorite`
- по умолчанию `false`

---

## 6. Правила для картинок

### Как должно быть
- одна картинка на рецепт
- хранится в `pages/recipes/img/`
- имя файла понятное и стабильное

### Именование
Использовать спокойные латинские имена:
- `chicken-broccoli-pie.jpg`
- `basque-cheesecake.jpg`
- `healthy-shawarma.jpg`

### Не делать
- не использовать случайные имена вроде `IMG_4837.JPG`
- не хранить изображения вне папки `img/`
- не использовать пробелы и кириллицу в имени файла

---

## 7. Что считается «добавили рецепт правильно»

Рецепт считается добавленным правильно, если:
- он появился в `recipes.json`
- картинка открывается
- рецепт находится через поиск
- рецепт виден в правильной категории
- модалка открывается без ошибок

---

## 8. Когда можно трогать `index.html`

`index.html` меняем только если нужно изменить:
- внешний вид каталога
- фильтры
- категории в интерфейсе
- отображение карточки
- отображение модалки
- общую логику поиска/сортировки

### Нельзя менять `index.html` только ради одного нового рецепта
Если задача звучит как:
> "добавить ещё один рецепт"

то почти всегда меняется только `recipes.json` и картинка.

---

## 9. Простой рабочий процесс на будущее

Когда Наталия присылает рецепт, процесс такой:

### Вариант A — есть полный рецепт
1. Привести текст к нормальному виду
2. Сохранить/подобрать картинку
3. Добавить объект в `recipes.json`
4. Проверить страницу

### Вариант B — есть скриншот или сырой текст
1. Сначала разобрать рецепт
2. Нормализовать поля
3. Подобрать категорию
4. Подобрать имя файла картинки
5. Только потом добавить в `recipes.json`

```json
    ""

```

---

## 11. Мини-правила, чтобы ничего не ломалось

- Не менять структуру JSON от рецепта к рецепту
- Не добавлять новые поля без необходимости
- Не править HTML, если добавляется только контент
- После добавления всегда проверять страницу глазами
- Если рецепт особенный — лучше обсудить, чем тихо ломать стандарт

---

## 12. Коротко

Для рецептов правило теперь такое:

**Новый рецепт = новая аккуратная запись в `recipes.json` + картинка в `img/`.**

Не правим страницу руками каждый раз. Не выдумываем новый формат. Не усложняем.

Это и есть нормальный, спокойный, поддерживаемый процесс.
