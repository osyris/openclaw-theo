# Price Tracker — отслеживание цен на товары

Единый детерминированный инструмент для отслеживания цен по ссылкам. Вся сложность
(обход бот-защит вроде Akamai/Cloudflare, разные магазины, рендер JS через локальный
headless-браузер chromium, извлечение цены, расписание) спрятана внутри скрипта.

**ГЛАВНОЕ ПРАВИЛО:** никогда не парси страницы, не угадывай цены и не редактируй
JSON/cron сам. Любое действие по трекингу цен — это вызов скрипта. Твоя роль:
перевести просьбу владельца в команду и доставить результат.

## Скрипт

`/data/workspace/skills/price-tracker/price_tracker.py` — запускай по полному пути
(`python3 /data/workspace/skills/price-tracker/price_tracker.py <команда>`).
Зависимости (curl_cffi, playwright) подтянутся сами через `uv`. Полный контракт:
команда `help`.

## Команды

| Просьба владельца | Команда |
|---|---|
| «следи за ценой на этой ссылке» | `… add "<url>" --title "<короткое имя>"` |
| «что ты отслеживаешь / список» | `… list` |
| «перестань следить за X» | `… remove <id>` |
| «приостанови / возобнови X» | `… disable <id>` / `… enable <id>` |
| «какие сейчас цены» / отчёт | `… check` → перешли владельцу stdout (сводка 🛍) **дословно** |
| страница не поддаётся, цену знаешь | `… set <id> <price>` |
| «всё ли работает» | `… doctor` (или `doctor --deep` — проверяет ещё и импорт playwright) |
| показать/настроить расписание | `… schedule` / `… schedule install` / `… schedule off` |

`id` берётся из вывода `list`.

## Поведение

- **add** сам находит рабочий способ получения цены и сразу пишет владельцу готовый
  результат: `✅ … добавил в трекер: <товар> — <цена>` либо `⚠️ … цену сразу вытащить
  не удалось …`. **Перешли этот текст владельцу как есть.** Цену не выдумывай.
- **check** печатает готовую сводку — перешли её как есть, без своих комментариев и
  переформатирования.
- JS-сайты рендерятся локальным chromium — внешних токенов/оплаты не нужно, отдельных
  предупреждений про токены больше нет.
- Ошибки скрипт печатает в stderr как `{"error":{...}}` — если команда вернула ошибку,
  объясни владельцу человеческим языком.

## Состояние и расписание (не трогать руками)

- Конфиг/история: `/data/workspace/skills/price-tracker/price-tracker.json` —
  меняется ТОЛЬКО командами выше.
- Расписание: проверка идёт автоматически **08:00 и 20:00 (Europe/Tallinn)** —
  это cron-джоба, которой управляет сам скрипт (`schedule install/off`). Cron руками
  не настраивай — только через `schedule`.
